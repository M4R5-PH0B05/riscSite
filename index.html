RISC vs CISC

What do RISC and CISC mean?

RISC and CISC are two different types of CPU architecure which both focus on seperate things. RISC stands for Reduced Instruction Set Computing, and has several key features, with the main ones being:

  • Simple, Fixed length instructions and single cycle instructions
  • Many general purpose registers rather than specific registers
  • The ability to use pipelining - which is the execution of multiple different instructions at the same time.
CISC stands for Complex Instruction Set Computing, and It's main features are as follows:
  • Complex, Variable length instructions which can take multiple cycles
  • Can operate directly on memory
  • Fewer registers which are more specialized

What are the Advantages / Disadvantages of RISC?

RISC is usually used with ARM (Advanced RISC machines) processors, which are commonly associated with mobile devices. They are also used in embedded systems, and high performance computing.

Advantages

  • Faster execution of instructions that are individual
  • The hardware design of the chip is simpler, and therefore easier to manufacture for companies.
  • It has a lower power consumption than CISC
Disadvantages
  • RISC processors may require more instructions in order to peform tasks that are more complex
  • The code that is written for these chips is most likely going to be longer
